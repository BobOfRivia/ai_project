# 6.8 可观测平台选型

> **核心问题**：LangSmith / Arize Phoenix / Braintrust / Langfuse / Helicone 在 2026 各占什么位置？什么团队选谁？
> **目标**：能根据团队栈（LangChain / 自研 / 多 provider）、上线节奏（开发期 / 生产期）、合规要求选对平台，并把"trace + eval + dataset + alerts"一站打通。

**展开重点**：
- **2026 三家拉开身位**：
  - **LangSmith**：LangChain / LangGraph 生态首选；2026-03 推出 **Sandboxes**（隔离执行环境）+ NVIDIA 合作；端到端 agent 平台（trace + eval + prompt management + dataset + 看板）；适合**已用 LangChain 栈**的团队
  - **Arize Phoenix**：**OTel 原生 + 开源 eval 库**；drift detection / embedding analysis 扎实；企业证书齐全（SOC2 / HIPAA），**合规行业首选**
  - **Braintrust**：**eval-driven 哲学**：observability 与 evaluation 一体；最像"AI 产品的质量管理系统"；CI/CD gates 友好；免费层 1M spans + 10K evals
- **第二梯队**：
  - **Langfuse**：2026-01 被 **ClickHouse 收购**；功能不变、社区活跃；**开源自托管首选**（无 per-seat、可私有部署）
  - **Helicone**：**最小摩擦 + gateway 能力**；改 BASE_URL 即可接入；适合"先有 trace 再说"的快速接入
  - **Latitude**：从 **agent session 为分析单位**设计；新势力
  - **MLflow / W&B Weave**：已经在 ML 平台上的团队自然延伸
- **维度对照**：
  - **trace 能力**：所有都有，差异在 UI / 查询 / OTel 兼容
  - **eval 集成**：LangSmith / Braintrust 一体化；Langfuse / Phoenix 有但偏 API；Helicone 弱
  - **prompt 管理 / version**：LangSmith / Braintrust / Langfuse 都有
  - **dataset 管理**：LangSmith / Braintrust 强；其他一般
  - **A/B 实验**：Braintrust / LangSmith 强
  - **CI 集成**：Braintrust 最深、LangSmith 紧随
  - **OTel 原生度**：Phoenix > Helicone > Langfuse > 其他
  - **自托管**：Langfuse 首选；Phoenix（OSS 部分）OK；其他多为 SaaS
  - **合规**：Phoenix / LangSmith / Braintrust 都有 SOC2；Phoenix 最齐
- **选型决策树**：
  1. 已经在 LangChain 上？→ **LangSmith**
  2. 强 OTel / 合规需求？→ **Arize Phoenix**
  3. eval 是核心、想 PR 闸门 + 完整数据流？→ **Braintrust**
  4. 强开源 / 自托管 / 无 per-seat？→ **Langfuse**
  5. 多 provider gateway / 快速接入？→ **Helicone**
- **常见多平台组合**：
  - 生产 tracing 走 Langfuse / Phoenix，offline eval 走 Braintrust
  - 内部 dev 走 LangSmith，线上走 OTel + 自建 Grafana
  - 通过 OTel GenAI（→ 6.7）让 trace 可移植
