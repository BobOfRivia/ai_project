# 3.12 Agent 工程化：Harness、评估与可观测

> **核心问题**：长时运行的 Agent 如何设计 Harness 基础设施？评估对象从"答案"变成"轨迹"后怎么评？生产中怎么观测和灰度？
>
> **目标**：
> - 掌握 Harness 设计：Brain / Hands / Session 解耦架构、Initializer + Coding Agent 双角色模式、跨 Session 状态桥接、Sandbox 隔离与凭证安全、AGENTS.md 约定
> - 掌握 Agent-specific 评估：pass@k vs pass^k、轨迹质量评分、工具调用准确率、评估套件生命周期管理
> - 掌握可观测：step-level tracing（LangSmith / Langfuse / Arize）、cost-latency profiling、失败模式分析
> - 掌握灰度上线：A/B、回归测试、人工复核触发策略
>
> **边界**：通用 eval / LLM-as-Judge → ch6；通用工程（路由 / cache / 部署） → ch7
