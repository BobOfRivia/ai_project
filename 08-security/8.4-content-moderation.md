# 8.4 内容审查与有害输出过滤

> **核心问题**：用户输入 / 模型输出审查应该用什么工具？OpenAI Moderation / Llama Guard 4 / Anthropic 各自适合哪种场景？怎么按 MLCommons 危害分类对齐？
> **目标**：能为应用设计"输入审查 + 输出审查 + 拒答模板 + 异常告警"的内容安全骨架，并把误拦率控制在产品可接受范围。

**展开重点**：
- **2026 主流审查方案**：
  - **OpenAI Moderation API**：免费、覆盖多类、与 GPT 系列对齐；适合英文为主、轻量集成
  - **Llama Guard 4**（Meta，12B 多模态）：开源、对齐 **MLCommons hazards taxonomy**、可微调、可本地部署；2026 默认开源选择
  - **Anthropic Claude（内置安全）**：行业领先的内置 safety，针对 Claude 调用时审查相对完善；但**外部应用层还需独立审查器**
  - **Azure Content Safety / AWS Bedrock Guardrails / Google Vertex Safety Filters**：云平台内置
- **MLCommons AI Safety v0.5+ 危害分类**（2026 事实标准）：
  - 暴力 / 性内容 / 仇恨 / 自伤 / 武器 / 违法活动 / 隐私侵犯 / 知识产权 / 错误信息 / CSAM
  - **Llama Guard 4 直接对齐这个分类**，比 OpenAI 分类更细
- **输入 vs 输出审查**（双向都要）：
  - **输入审查**：拦截恶意用户、合规要求（CSAM 必拦）、PII 检测（→ 8.6）
  - **输出审查**：防模型生成有害 / 泄漏 / 偏见内容；多模态生成（图 / 视频）必加
  - 实务：输入审查可较松（成本控制），输出审查从严
- **审查器组合架构**：
  - 流水线：用户输入 → 输入审查 → LLM → 输出审查 → 用户
  - 双审查器：用 A 模型生成、用 B 模型审；不同 family 减少同质偏见
  - 多模态：图 / 视频 / 音频要专门审（Llama Guard 4 多模态版 / 商业 API）
- **审查精度与误拦**：
  - **Llama Guard 在 ToxicChat 上超过 GPT-4**，开源场景首选
  - **误拦率 (FPR)** 关键指标：产品角度 FPR > 2-5% 用户体验已经不行
  - 阈值要按业务调：客服可严、创作工具要松
- **拒答模板与降级**：
  - 拒答要分类型：合规拒答（不可改）/ 偏好拒答（可让用户重述）/ 安全拒答（提供帮助资源，如自伤场景）
  - 不要单一"作为 AI 我不能..."，体验差也是产品缺陷
- **审查的工程边界**：
  - 多语言覆盖（Llama Guard 已多语言，OpenAI Moderation 英语为主）
  - **混淆攻击**：base64 / unicode / 拼写变体；要做 normalization
  - **流式输出**审查：边生成边审，命中即截断 / 重写
  - **延迟开销**：Llama Guard 7-12B 本地推理 ~100-300ms；OpenAI Moderation ~50-150ms
- **常见反模式**：
  - 只审用户输入不审模型输出
  - 把审查放到主模型调用之后才返回——延迟翻倍，流式体验差
  - 不监控拦截率：突涨可能是攻击 / 突跌可能是审查器挂了
  - 用同一个模型既生成又审查：同源偏差
  - 多模态产品只审文本
