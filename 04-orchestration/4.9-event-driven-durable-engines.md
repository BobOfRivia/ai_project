# 4.9 事件驱动与 Durable Execution 引擎

> **核心问题**：Temporal / Inngest / Restate / Hatchet / DBOS / Prefect 把 LLM workflow 当 durable function 跑的路线是什么？什么时候该上 Temporal 而不是 LangGraph？
> **目标**：能识别"业务级 durable workflow"需求（跨天、跨服务、强一致），并在 LLM 原生框架与传统 workflow 引擎之间选对路线。

**展开重点**：
- **2026 行业判断**：durable execution 从 niche 变成 **production AI 标配**；LLM 调用 / 工具 / agent 步骤天然 non-deterministic，必须放进 Activity 才能让 orchestration 可靠
- **Temporal**：2026-02 拿 $300M D 轮、$5B 估值；OpenAI / Replit / Lovable 都在上面跑 agent；Replay 2026 推出 serverless workers + workflow streams
- **Inngest**：事件驱动 + AgentKit；"从现有 app 代码到 durable workflow 最短路径"
- **Restate**：durable function，service-boundary 级别（不仅是后台任务）
- **Hatchet**：同档新势力
- **DBOS**：Postgres-backed durable execution；与 Temporal 直接竞争
- **Prefect**：数据 workflow 出身，LLM 友好
- **核心抽象**：**确定性回放**（deterministic replay）—— 把"业务编排"和"非确定性 LLM 调用"分层
- **LangGraph vs Temporal 决策线**：
  - 单进程内 / 单团队 / LLM 为主 → LangGraph
  - 跨服务 / 跨天 / 业务级 SLA / 多语言 → Temporal / Restate
  - 已有 Inngest / Temporal 业务栈 → 直接复用
