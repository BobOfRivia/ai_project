# 6.10 CI 闸门与线上监控

> **核心问题**：PR 闸门怎么搭？A/B 怎么跑？线上 drift / refusal / retry / cost 怎么监控？什么时候自动回滚？
> **目标**：能让团队的 prompt / 模型 / 检索配置变更都走"评估闸门 + 线上观测 + 自动告警"，把"上线后才发现质量回退"变成"PR 阶段就拦住"。

**展开重点**：
- **CI 闸门标准做法**（2026 共识）：
  - PR 触碰 prompt / 模型 / retrieval config → CI 跑 golden set
  - **质量回归 → 阻塞合并**（"quality gate"）
  - **绿灯报告**附在 PR 上：哪些指标 / 涨多少 / 跌多少 / 失败例
  - **必跑 + 选跑**：必跑（核心指标 + 关键例）几分钟；选跑（全套）几十分钟
  - **A/B prompt / 模型**：同一 dataset 双跑、看差异
  - 平台支持：**Braintrust / LangSmith / Confident AI** 把这套做成开箱即用
- **统计严谨度**：
  - 比较两版本必须看置信区间、p-value、效应大小
  - 样本量太小（< 30）不要相信均值差异
  - 失败例（regression items）要 list 出来，不要只看汇总
- **A/B 上线实验**：
  - 灰度（feature flag）+ 用户分桶
  - 多臂：control / treatment_A / treatment_B
  - 评估指标：质量（judge / user thumb）+ 业务（转化 / 留存）+ 体感（latency / cost）
  - **统计显著 ≠ 业务重要**：注意效应大小
- **线上监控核心维度**：
  - **质量漂移（drift）**：online judge 分数随时间下降；输入分布偏离训练 / golden set
  - **Refusal rate**：拒答比例突变（模型升级 / safety 调严最常引发）
  - **Retry rate**：函数调用 / JSON parse 重试上涨 → 工具 schema 不对或模型变了
  - **Latency P50/P90/P99**：分模型 / 分路径
  - **Cost per request / per task**：模型路由 / cache miss / token 膨胀
  - **Tool error rate**：agent 工具失败率
  - **Hallucination signals**：citation broken / faithfulness 低分
- **告警与自动响应**：
  - 关键指标设阈值 + slope（突变检测）
  - 告警分级：page（紧急回滚）/ ticket（次日）/ 日报（仅记录）
  - **自动回滚**：A/B 实验里 control 显著好 → 自动收流量
  - **降级链**：模型不可用 → 备用模型 → 降级到便宜模型 → 友好失败
- **运营节奏**：
  - 周度：看核心质量 / 业务指标趋势
  - 月度：judge 与人评对齐
  - 季度：golden set audit、hold-out 更新
- **常见反模式**：
  - 只看 latency / cost，不看质量（"上线就放手"）
  - 评估闸门设太严 / 太多，团队绕过
  - online judge 直接用同一个生产模型 → self-evaluation 偏差
