# 2.3.1 Embedding 模型是怎么训练的（深入 · 选读）

> **定位**：本章正文只讲 embedding 的**选型与使用**（2.3）。本篇是**深入选读**，讲"这些模型是怎么训练出来的、在 LLM 基础上改了什么"。属算法工程师范畴——应用工程师**按需读**，但读懂后能解释：为什么要用 `query:`/`passage:` 前缀、为什么指令型模型要给对指令、为什么能 Matryoshka 截断、什么时候值得在领域数据上微调 embedding。
> **核心问题**：从一个生成式 LLM 到一个能算语义距离的 embedding 模型，中间发生了什么？主流训练管线是什么？

---

## 一、目标的根本转变：从"生成"到"表示"

先看 LLM 和 embedding 模型在**训练目标**上的本质区别：

| | 生成式 LLM | Embedding 模型 |
|---|---|---|
| 目标 | **预测下一个 token**（生成） | 把整段文本压成**一个向量**，让语义近的文本向量也近（表示） |
| 输出 | token 概率分布 | 一个 $d$ 维稠密向量 |
| 训练信号 | 交叉熵（预测对不对） | **对比学习**：正样本对拉近、负样本推远 |
| 注意力 | 因果（只看左侧） | 通常要**双向**（看全文语境） |

一句话：**LLM 学"接下去说什么"，embedding 学"这段话整体是什么意思、和别的话有多像"。** 训练 embedding 就是把一个模型从前者改造成后者。

**两条技术路线：**

- **编码器路线（经典）**：以 BERT 类双向编码器为骨干，Sentence-BERT（SBERT）开创了用对比/孪生网络训句向量。BGE、E5、GTE 早期都属这条。
- **LLM-based 路线（2024–2026 主流上分）**：直接拿**解码器 LLM**（Mistral、Qwen 等）当骨干改造成 embedding。代表 E5-mistral、NV-Embed、GTE-Qwen、Qwen3-Embedding。榜单头部现在多是这条路线。

---

## 二、在 LLM 基础上做的三个结构性改造

把一个 decoder-only LLM 变成 embedding 模型，主要改三处：

### 2.1 Pooling：怎么从 token 序列得到"一个"向量

模型对每个 token 输出一个隐藏向量，但我们要**整句一个向量**，得做池化：

- **Mean pooling（均值池化）**：所有 token 向量取平均。编码器路线常用。
- **[CLS] pooling**：取特殊起始 token 的向量（BERT 传统做法）。
- **Last-token / \<EOS\> pooling**：取**序列最后一个 token（常是 \<EOS\>）的末层隐藏态**。decoder-only LLM 因果注意力下，最后一个 token"看过"全文，所以用它——E5-mistral、Qwen 系常用。
- **Latent Attention pooling（NV-Embed）**：额外加一个可学习的注意力池化层，比简单均值/末 token 更好地聚合信息。

### 2.2 注意力：因果 → 双向

decoder-only LLM 是**因果注意力**（每个 token 只能看左边），但**做表示需要双向语境**（"苹果"是水果还是公司要看右边的词）。所以：

- **LLM2Vec、GTE-Qwen 等把因果掩码换成 BERT 式双向注意力**再抽 embedding。
- 这是 LLM-based embedding 的关键一步——**解锁双向语境**。

### 2.3 加指令 / 前缀（Instruction / Prefix）

现代 embedding 是**指令感知**的，训练时就把任务写进输入：

- **非对称前缀**：查询加 `query:`、文档加 `passage:`（E5 系），让模型区分"提问方"和"被检索方"——**这就是你用时必须加对前缀的原因**，训练分布就是这么定的。
- **任务指令**：如 `"Given a web search query, retrieve relevant passages: {query}"`。E5-mistral、NV-Embed 用指令统一多种任务（检索/分类/聚类/STS）到一个模型。

> ⚠️ 应用侧直接后果：**用错前缀/指令会掉点**。选了指令型模型，就要按它文档给的模板拼输入。

## 三、核心训练方法：对比学习（Contrastive Learning）

这是 embedding 训练的心脏。

### 3.1 思想

准备**正样本对**（语义相关的两段文本，如"问题—答案""标题—正文"）和**负样本**（不相关的文本）。训练目标：**让正样本对的向量相似度高、负样本对的相似度低**。

### 3.2 InfoNCE 损失

主流损失是 **InfoNCE**。对一个查询 $q$、它的正样本 $p^+$、以及一批负样本 $\{p^-_1, \dots, p^-_n\}$：

$$L = -\log \frac{\exp\!\big(\text{sim}(q, p^+)/\tau\big)}{\exp\!\big(\text{sim}(q, p^+)/\tau\big) + \sum_{i=1}^{n} \exp\!\big(\text{sim}(q, p^-_i)/\tau\big)}$$

- $\text{sim}(\cdot,\cdot)$：通常是**余弦相似度**（向量先归一化）。
- $\tau$：**温度**，控制分布尖锐程度，是关键超参。
- 直觉：这其实是一个"从一堆候选里把正样本挑出来"的**softmax 分类**——分子是正样本、分母是正+所有负样本。最小化 $L$ = 让模型把相似度质量集中到正样本上。

### 3.3 负样本从哪来（决定质量的关键）

- **In-batch negatives（批内负样本）**：同一个 batch 里**其他样本的正文档**直接拿来当负样本——几乎零成本，所以 embedding 训练**偏爱超大 batch**（负样本越多，对比信号越强）。
- **Cross-batch / cached negatives（跨批缓存负样本）**：缓存最近几个 batch 的向量当额外负样本，进一步扩大负样本池。
- **Hard negatives（硬负样本）**：**语义相近但实际不相关**的样本——最难、最有用（见第五节）。

## 四、主流训练管线：多阶段

2026 年头部 embedding 模型基本都是**多阶段渐进式训练**：

```
Stage 0（可选）：骨干改造
  decoder LLM → 去因果掩码/双向化 + 池化层  (LLM-based 路线)
        │
        ▼
Stage 1：弱监督对比预训练
  海量、带噪的文本对（网页 title-body、QA、爬取数据）
  用 InfoNCE + in-batch negatives，学通用语义
        │
        ▼
Stage 2：监督微调
  较小、高质量的人工标注对 + 挖掘的 hard negatives
  精修检索相关能力（这一步最提检索质量）
        │
        ▼
Stage 3（可选）：蒸馏 / 软信号
  用 cross-encoder（2.6 的重排器）的 listwise 打分做知识蒸馏
  让 bi-encoder 学到更细粒度的相关性
        │
        ▼
（叠加）Matryoshka：训练时在多截断维度上联合算损失 → 支持按需截断（见 2.3）
```

- **Stage 1（弱监督预训练）**：数据量大但脏（弱监督），用海量正样本对 + 批内负样本打好通用语义底子。
- **Stage 2（监督微调）**：数据少而精（人工标注/精挖），加 **hard negatives**，是**检索质量的主要来源**。
- **Stage 3（蒸馏）**：用更强的 cross-encoder 当老师，把它的排序知识蒸给快速的 bi-encoder——"用重排器的质量喂检索器"。

## 五、硬负样本挖掘（Hard Negative Mining）—— 质量分水岭

普通的批内负样本大多"一眼不相关"，模型很容易分对，学不到细粒度差异。**Hard negative = 长得像正样本、其实不相关**，逼模型学会精细区分——这是 Stage 2 提分的核心。

**怎么挖：**

1. 用一个已有检索器（甚至上一轮的自己）对每个 query 取 top-k。
2. 这些 top-k 里，**非正样本的那些**就是好的 hard negatives（它们"排得上来"说明够像）。
3. ⚠️ **阈值过滤防"假负样本"**：太靠前的可能其实是**没标注的正样本**，误当负样本会伤害训练——用相似度阈值卡掉过于接近正样本的。

**进阶**：**动态 / 在线硬负样本挖掘**——训练中持续更新 hard negatives，而不是训练前一次挖定；也有用 LLM **合成** hard negatives（改写、增强）的做法。

## 六、LLM-based embedding 的现代范式（2024–2026）

代表工作及其关键手法，帮你把上面拼起来：

| 模型 | 骨干 | 关键手法 |
|---|---|---|
| **E5-mistral** | Mistral-7B | **GPT-4 合成海量训练对** + 指令；末 token 池化；超越同期双向模型 |
| **LLM2Vec** | 各种 decoder LLM | 三步把 decoder 转 encoder：**双向注意力** + 掩码下一 token 预测（MNTP）+ 无监督对比；只用公开数据 |
| **NV-Embed** | decoder LLM | **Latent attention 池化** + 去因果掩码 + 两阶段训练；通用型 SOTA |
| **GTE-Qwen / Qwen3-Embedding** | Qwen | 双向化 + 多阶段 + 多语言；中文强 |
| **BGE-M3** | 编码器路线 | 一模型同出 dense/sparse/多向量，多语言（本库 2.3 自托管默认） |

**两个贯穿的趋势：**

1. **合成数据**：用强 LLM（GPT-4 类）批量造 query-文档对和 hard negatives，突破人工标注的数据瓶颈。
2. **指令统一多任务**：一个模型靠不同指令同时干检索/分类/聚类/STS，不再一任务一模型。

## 七、对应用工程师的意义（takeaways）

即使不自己训练，理解上面能帮你用好、选好 embedding：

1. **前缀/指令必须用对**：`query:`/`passage:` 和任务指令是**训练时就固定的分布**，用错直接掉点——照模型文档的模板拼输入。
2. **Matryoshka 能截断**是因为训练时叠了多截断损失（见 2.3 专栏），不是所有模型都行。
3. **看训练数据的语言/领域覆盖**：中文场景选中文语料训练充分的（BGE-M3、Qwen3-Embedding），别用纯英文模型硬跑。
4. **领域微调值不值**：通用模型在垂直领域（法律、医疗、代码）可能不够；有高质量领域正样本对时，**在通用模型上做 Stage 2 式对比微调 + hard negatives** 常能明显提分——但要权衡数据标注和训练成本（回到本库"落地性"原则，先确认通用模型 + 好检索策略真的不够再上）。
5. **hard negatives 的价值**：这解释了为什么"看起来相近的干扰项"是检索难点——模型训练时就在专门对付它们。

> 一句话收束：**embedding 模型 = 把 LLM 从"生成器"改造成"语义标尺"**——改池化、改注意力、加指令，再用对比学习（InfoNCE + hard negatives）分阶段训出来。用的时候尊重它的训练约定（前缀/指令/维度），就能发挥出应有的检索质量。
