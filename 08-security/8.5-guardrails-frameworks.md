# 8.5 Guardrails 框架与产品化

> **核心问题**：NeMo Guardrails / Guardrails AI / LLM Guard 三家在 2026 各占什么位置？怎么组合落地"输入 / 流程 / 输出"三道关？
> **目标**：能选定 1-2 个 Guardrails 框架，把 8.2-8.4 / 8.6 的能力（注入检测 / 内容审查 / PII / 输出 schema 校验）做成产品级中间件。

**展开重点**：
- **Guardrails 定位**：
  - 是"输入 / 流程 / 输出"中间件，不替代审查模型本身——它**编排和声明**审查策略
  - 与 7.5 LLM Gateway 互补：Gateway 管 provider 路由 / cost，Guardrails 管内容与合规
- **2026 三大框架对比**：
  | 框架 | 语言 | 特点 | 适用 |
  |---|---|---|---|
  | **NeMo Guardrails** (NVIDIA) | Python，**Colang DSL** | GPU 加速、5 阶段管线、<100ms、对话流程控制强 | 复杂对话 Agent，需要流程级安全策略 |
  | **Guardrails AI** | Py + **JS** | **可组合 validator**，hub 上 100+ 现成 validator，freemium | API/JSON 输出结构化校验，跨语言团队 |
  | **LLM Guard** (Protect AI) | Python | 开源、输入 + 输出两套 scanner（注入 / PII / 毒性 / 越权） | 单点拦截，与 LiteLLM / Portkey 集成 |
- **NeMo Guardrails 5 阶段管线**：
  1. **Input rails**：用户输入审查
  2. **Dialog rails**：对话流程控制（Colang 定义"用户问 X 时只能走 Y 流程"）
  3. **Retrieval rails**：RAG 检索内容审查（防 indirect injection）
  4. **Execution rails**：工具调用前后检查
  5. **Output rails**：输出审查
- **Guardrails AI 典型 validator**：
  - `RestrictToTopic`（限定话题）/ `PIIFilter` / `ToxicLanguage` / `CompetitorCheck` / `RegexMatch` / `ValidJSON` / `ProvenanceLLM`（事实出处校验）
  - **Re-ask 模式**：检测失败时自动 re-ask LLM 修复（vs 直接拒）
- **LLM Guard scanner（输入 + 输出）**：
  - 输入：`Anonymize`、`PromptInjection`、`TokenLimit`、`Toxicity`、`BanTopics`、`Code`
  - 输出：`Deanonymize`、`NoRefusal`、`Sensitive`、`URLReachability`、`Bias`、`MaliciousURLs`
- **组合落地模式**（业界常见）：
  - **NeMo + Guardrails AI**：NeMo 管对话流程、Guardrails AI 管输出 schema
  - **LiteLLM + LLM Guard**：网关层嵌 LLM Guard scanner（→ 7.5）
  - **Portkey + Guardrails AI**：Portkey 做 routing / cost、Guardrails AI 做 schema 校验
- **延迟与成本预算**：
  - 输入审查 ≤ 100ms 是产品级目标
  - 不要让 Guardrails 把流式 TTFT 拉爆——能并行的并行
  - **多 validator 串行 vs 并行**：独立 validator 并行；有依赖（先解码再扫毒）才串行
- **可测试性 / 可演化**：
  - Guardrails 规则要纳入 Git + Code Review
  - 每条规则有**单元测试**（已知攻击 + 已知正常）
  - 上线前用 Promptfoo / Garak 跑回归（→ 8.8）
- **常见反模式**：
  - "上 NeMo 就安全了"——Guardrails 是骨架不是子弹
  - 规则改了不测试就上线：误拦率突涨
  - 输出审查发现问题直接拒答——优先尝试 re-ask 修复
  - 把所有 validator 都开——延迟和成本双爆
  - 没监控 Guardrails 自己挂了：审查服务挂 = 应用裸奔
