# 8.1 安全与合规基础与威胁模型

> **核心问题**：LLM 应用的攻击面到底有几个？应该用什么思维框架而不是被"AI 安全"这种宽泛词糊弄？
> **目标**：能用 OWASP LLM Top 10 v2 + 4 层威胁模型把任意 LLM 应用的风险点点出来，并理解 "blast radius (爆炸半径)" 思维。

**展开重点**：
- **OWASP LLM Top 10 v2（2025/2026 现行版）十大风险**：
  1. **LLM01 Prompt Injection**（永远第一）
  2. **LLM02 Sensitive Information Disclosure**（系统提示泄漏、训练数据泄漏、用户数据泄漏）
  3. **LLM03 Supply Chain**（模型 / 数据集 / 适配器 / 插件被投毒）
  4. **LLM04 Data and Model Poisoning**（训练 / 微调 / RAG 索引污染）
  5. **LLM05 Improper Output Handling**（直接把模型输出注入 SQL / shell / HTML）
  6. **LLM06 Excessive Agency**（工具权限超出业务需要）
  7. **LLM07 System Prompt Leakage**
  8. **LLM08 Vector and Embedding Weaknesses**（RAG 注入 / 跨用户隔离失败）
  9. **LLM09 Misinformation**（幻觉作为安全事件）
  10. **LLM10 Unbounded Consumption**（DoW = Denial of Wallet / token 烧光）
- **4 层威胁模型**（适配 7 章的 4 层视角）：
  - **输入层**：用户输入、RAG 检索内容、工具返回、上游 webhook
  - **模型层**：模型本身（幻觉、有害输出、提示泄漏）
  - **动作层**：工具调用、数据库写、shell 执行
  - **输出层**：返回给用户、写入下游、被另一个 Agent 消费
- **"Blast Radius" 思维**（2026 共识）：
  - **接受注入会发生**——目标不是"100% 防住"而是**出事时影响范围最小**
  - 三原则：**Least Privilege**（最小权限）/ **Sandbox**（执行隔离）/ **Auditability**（每个动作可追溯）
  - 单层控件都能被绕过、**多层组合才有效**
- **威胁建模实操**：
  - 画出 trust boundary（信任边界）：哪些输入来自不可信源？
  - 标记每个动作的 reversibility（可逆性）：发邮件、写数据库是不可逆
  - 对不可逆动作要求二次确认 / human-in-the-loop
- **安全 SLO**（与 7.1 SLO 三件套互补）：
  - **MTTD / MTTR**（事件检测 / 响应时间）
  - **Red Team pass rate**（CI 里的红队测试通过率）
  - **Guardrail block rate**（拦截率，过高也是信号）
  - **Audit log 完整率**
- **常见反模式**：
  - "我们用了 OpenAI，安全是他们的事"——错，应用层漏洞 100% 在你这边
  - 把模型输出当可信源直接 eval / SQL
  - 一次性渗透测试当做"安全做过了"，不进 CI
  - 把"合规"和"安全"混为一谈：合规过线不等于安全
