# 4.8 低代码 / 可视化编排平台

> **核心问题**：Dify / Coze / n8n / Flowise / FastGPT / Open WebUI / Botpress / RAGFlow 各自的位置是什么？什么时候用平台，什么时候自建代码？
> **目标**：能判断"PoC 用 Dify、生产改自建"这类常见路径，从迭代速度、控制粒度、团队角色、成本、数据合规等维度做出决策。

**展开重点**：
- **Dify**：开源 LLMOps + 商业云，2026 已 50k+ stars；Backend-as-a-Service + 工作流编排合体
- **Coze**：字节系，**全球版 + 国内版**双产品，强在"多渠道发布（Telegram / Discord / 微信 / API）"与字节生态
- **n8n**：通用 workflow 老牌玩家，内置 LangChain Agent 节点，可塞 JS / Python，工程师友好
- **Flowise**：LangChain 可视化编辑器，开发者向
- **FastGPT**：RAG 强，中文场景常用
- **Open WebUI / Botpress / RAGFlow**：同档但定位略偏（聊天 UI / 客服 bot / RAG 引擎）
- **决策维度**：
  - 迭代速度（平台快 1 个数量级）
  - 控制粒度（自建无上限）
  - 团队角色（业务能用平台，工程师必要时下到代码）
  - 数据合规（自建可私有部署、平台看版本）
  - 多渠道发布（Coze 强项）
  - 退出成本（平台锁定 vs 代码可迁移）
