# 8.9 合规法规速览（EU AI Act / NIST / GDPR）

> **核心问题**：作为应用工程师，需要知道哪些法规、哪些条款会落到代码 / 流程 / 文档要求里？什么时间点钉死？罚则是多少？
> **目标**：能在产品讨论里指出"这个 feature 触发 X 条款 / 我们需要 Y 文档 / Z 控件"，让法务和工程对齐而不是后期推翻。

**展开重点**：
- **2026 必懂三大支柱**：
  - **EU AI Act**（欧盟，全球最严，外溢效应大）
  - **NIST AI RMF + AI 600-1 GenAI Profile**（美国，事实门槛，联邦采购引用）
  - **GDPR**（欧盟数据保护，LLM 应用普遍触发）
- **EU AI Act 时间线钉死**：
  | 日期 | 生效条款 |
  |---|---|
  | 2025-02-02 | 不可接受风险 AI 禁令（social scoring、subliminal manipulation） |
  | 2025-08-02 | **GPAI 提供者义务**（透明、文档、版权、训练数据摘要） |
  | **2026-08-02** | **Commission 执法权启动 + 多数高风险义务生效** |
  | 2027-08-02 | 高风险 AI 全部条款生效 |
- **EU AI Act 风险分级**（工程师视角）：
  - **不可接受**：禁止
  - **高风险**（Annex III：HR、教育、信贷、执法）：CE 标识、风险管理、数据治理、人工监督、登记 EU 数据库
  - **有限风险**（chatbot、deepfake）：透明告知（"你在和 AI 对话"）
  - **最小风险**：无义务
- **GPAI 提供者义务**（model provider，应用层引用时知道下游有什么要求）：
  - 技术文档（架构、训练）
  - 下游开发者支持文档
  - 版权政策 + EU 版权法对齐
  - 训练数据摘要（足够详细）
  - **Systemic risk 模型**（>10^25 FLOPs）：额外报告 + 评估义务
  - **罚则**：最高 €15M 或全球营收 3%（高风险违规更高，€35M 或 7%）
- **NIST AI RMF + AI 600-1 GenAI Profile**：
  - 4 大功能：**GOVERN / MAP / MEASURE / MANAGE**
  - GenAI Profile 列出 **12 风险类**：confabulation（幻觉）、prompt injection、训练数据隐私、有害内容、CBRN 信息、价值链复杂性 等
  - 美国联邦采购 RFP 普遍引用（虽非强制法律）
  - 配合 **NIST AI 100-2 / 100-3 / 800-218A SSDF for AI** 一起看
- **GDPR 对 LLM 应用的核心命中**：
  - **法律基础**：合同 / 合法利益 / 同意
  - **数据主体权利**：访问 / 删除（LLM memory / RAG 索引要支持）
  - **跨境传输**：充分性决定 / SCC / BCR
  - **自动化决策**（Article 22）：信贷 / HR 等场景需人工监督
  - **DPIA**（数据保护影响评估）：高风险数据处理必做
  - **罚则**：€20M 或全球营收 4%
- **美国联邦 + 州级速览**：
  - **联邦**：Biden 2023 EO 14110 → Trump 2025 EO 14179（撤销大部分），各部门规则仍有效
  - **EEOC**（雇佣歧视）、**FTC**（不公平做法）、**HIPAA**（医疗）、**FCRA**（信贷）
  - **加州**：CCPA/CPRA、AB 2013（训练数据透明）、SB 1047（曾被否决，2026 重审）
  - **科罗拉多 AI Act 2024**：2026-02 生效，类似 EU Act 缩小版
- **其他重点辖区**：
  - **中国**：生成式 AI 服务管理暂行办法（2023-08）、深度合成规定（2023-01）、PIPL
  - **英国**：原则导向（5 大原则），未单独立法
  - **加拿大**：AIDA（C-27 中），延期
  - **巴西 / 印度 / 韩国**：本地化 AI 法案推进中
- **工程层面的"合规清单"**（最小集）：
  - 用户告知（你在和 AI 对话）
  - PII 处理与脱敏（→ 8.6）
  - 训练数据 / Prompt 不留版权问题
  - 审计日志保留期符合监管
  - 高风险场景 HITL
  - 模型 / Prompt 版本可追溯
  - DPIA / Conformity Assessment 文档
- **常见反模式**：
  - "我们没在欧盟"——只要服务欧盟用户就触发 EU AI Act
  - 把合规留到法务最后 review：架构已经定型再改成本巨大
  - 把"用了某云"当合规：云厂的合规是基础设施层，不等于你的应用合规
  - 不留训练 / 微调数据来源记录：被问起说不清
