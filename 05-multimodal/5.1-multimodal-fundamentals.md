# 5.1 多模态基础与决策框架

> **核心问题**：什么是 2026 的"多模态应用"？VLM、生成模型、语音模型在一个应用里怎么协作？什么时候该用通用 VLM，什么时候该上专门模型？
> **目标**：面对一个新需求时，能在"模态 × 任务 × 模型"的矩阵上点出位置，并落到**确定性骨架 + 定点 multimodal 调用**的架构上。

**展开重点**：
- **多模态四象限**：理解（input：VLM / ASR / 视频理解）vs 生成（output：图像生成 / TTS / 视频生成）× 同步（streaming / realtime）vs 异步（batch）
- **2026 共识架构**（延续 ch4）：**确定性骨架编排 + 在确定步骤上插 multimodal 调用**；不要让 VLM 当总指挥（除非是 Computer Use Agent 类场景，归 3.10）
- **通用 VLM vs 专门模型**：
  - 通用 VLM（GPT-5 / Claude / Gemini）：OCR、文档智能、Visual QA、图像描述、Computer Use 视觉端 —— 多数任务上"够用"已成事实
  - 专门模型：高精度 OCR（PaddleOCR / Surya）、医学影像、卫星图等垂域；视频原生模型（Twelve Labs Marengo / Pegasus）
- **模态融合 vs 模态切换**：是同一个调用同时吃多模态（GPT-5 多模态输入），还是用模态间专门模型链式处理（ASR → LLM → TTS）
- **决策矩阵**：
  - 模态数量（单 / 双 / 三模态）
  - 延迟需求（batch / 流式 / realtime）
  - 精度需求（通用够用 / 垂域必须专模）
  - 成本预算（多模态 token 是文本的 5–20×；视频更甚）
  - 数据合规（云 API / 私有部署 / 边缘）
  - 可控性（生成模型的指令遵循 / 风格一致性）
- **常见组合范式**：
  - 文档智能：上传 PDF → VLM 解析（5.2）→ 结构化输出 → 后续 RAG（ch2）
  - 多模态客服：语音输入（5.5）→ 编排（ch4）→ 工具（ch3）→ 语音输出（5.6/5.7）
  - 内容创作：文本 brief → 图像生成（5.3）→ 编辑迭代（5.4）→ 视频生成（5.9）
  - 视频审核：抽帧 + VLM（5.8）或视频原生模型（Twelve Labs）
