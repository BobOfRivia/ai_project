# 8.8 LLM Red Teaming 与安全测试

> **核心问题**：怎么把 Prompt 注入 / 越狱 / PII 泄漏 / 工具滥用做成可重复的自动化测试？Garak / Promptfoo / PyRIT 各占什么位置？
> **目标**：能搭一条"上线前 + 持续回归"的安全测试流水线，把 Red Team 结果当 CI gate，和 6 章 Eval 共用 trace 基础设施。

**展开重点**：
- **2026 Red Teaming 三剑客**：
  | 工具 | 出品 | 强项 | 测试范围 |
  |---|---|---|---|
  | **Garak** | NVIDIA, Apache 2.0 | **120+ probe 模块**，纯模型 endpoint 测试 | 探针型，针对模型本身（注入、毒性、泄漏、编码绕过） |
  | **Promptfoo** | OpenAI 旗下（2026-03 收购 $86M），仍 MIT | **50+ 漏洞类型**、RAG / Agent / 工具链整体测试、CI 集成强 | 应用级 + breadth |
  | **PyRIT** | Microsoft | **多轮对话攻击**、思维链型攻击 | 深度型，模拟真实攻击者 |
  - 业界共识：**Promptfoo (breadth) + PyRIT (depth) + Garak (probes)** 组合用
- **Promptfoo 应用级测试能力**：
  - 一份 YAML 描述应用 → 自动跑 50+ 类攻击 → 报告
  - 覆盖：注入、越狱、PII 泄漏、有害输出、过度代理、Hallucination、工具滥用、RAG 注入
  - 对齐 **OWASP LLM Top 10** / **NIST AI RMF** / **MITRE ATLAS**
  - **CI 集成**：GitHub Actions / GitLab CI gate，pass rate < 阈值则 block
- **Garak probe 类型示例**：
  - `dan`（DAN 越狱）、`promptinject`、`encoding`（base64/ROT13/unicode）、`leakreplay`（训练数据泄漏）、`malwaregen`、`continuation`、`packagehallucination`（幻觉包名）
- **PyRIT 强项**：
  - **多轮 adversarial conversation**：模拟逐步诱导
  - 与 Azure OpenAI 集成最深
  - 可定制 attack strategy / scoring
- **测试节奏**（2026 最佳实践）：
  - **每个 PR**：跑轻量 Promptfoo 套件（< 5 min）
  - **每次模型 / Prompt 升级**：跑全量回归（Promptfoo + Garak）
  - **每月**：PyRIT 多轮深度 + 人工红队
  - **每季**：第三方专业红队（监管行业必做）
- **覆盖矩阵**（必测维度）：
  - 直接注入 / 间接注入（RAG / 工具返回 / 浏览内容）
  - 越狱（角色扮演 / 编码 / 多语言）
  - PII 泄漏（系统提示泄漏、训练数据泄漏、跨用户泄漏）
  - 有害内容生成（按 MLCommons 分类）
  - 工具滥用（→ 8.3）
  - DoW（→ 7.10，无限消耗）
- **结果分析**：
  - **Pass rate**：通过率（>95% 才算可上线，行业实操）
  - **Severity 分级**：Critical / High / Medium / Low
  - **Trend**：每个版本的趋势比绝对值更有意义
  - **False positive 标注**：Red Team 工具会误报，要持续标注
- **与 Eval（6 章）共存**：
  - Eval = 质量回归；Red Team = 安全回归
  - 共用 trace 基础设施（LangSmith / Langfuse / Braintrust）
  - 同一份 dataset 可双用（安全 case + 质量 case）
- **常见反模式**：
  - 只在上线前测一次：模型 / Prompt 一改全失效
  - 不更新 attack 库：新型注入半年内不断涌现
  - 只测主语言（英文），多语言绕过通行
  - 不测 Agent 工具链：只测单 LLM call 不代表 Agent 安全
  - Pass rate 当唯一指标：忽略 severity 与 trend
