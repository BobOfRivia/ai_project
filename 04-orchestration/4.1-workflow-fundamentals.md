# 4.1 工作流基础与决策框架

> **核心问题**：什么是 LLM workflow？Workflow vs Agent 的分水岭怎么落地？Anthropic 五类 workflow pattern 怎么选？**2026 的"确定性骨架 + 定点 agent"**架构是什么？
> **目标**：在面对一个新任务时，能给出"用 workflow 还是 agent，哪类 pattern"的判断；并能用 backbone-and-agent 模式把复杂应用拆成可控结构。

**展开重点**：
- Workflow 定义：predetermined code paths
- Agent 定义：LLM 动态决定自身流程与工具使用
- Anthropic 五类 pattern：Prompt Chaining / Routing / Parallelization（Sectioning + Voting）/ Orchestrator-Workers / Evaluator-Optimizer
- **2026 共识架构**："deterministic backbone with intelligence deployed at specific steps; agents invoked intentionally by the flow, control returns to backbone on completion"
- 决策矩阵：确定性 vs 自主性 / 可观测 vs 灵活性 / 成本 vs 能力 / 失败可追溯性
