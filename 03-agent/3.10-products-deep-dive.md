# 3.10（深拆一）Computer Use / Browser Use 头部产品逐个拆解

> **本文定位**：3.10 主文件讲"路线与方法论"，本文专讲"**市面头部产品到底是什么、怎么实现的、差异化在哪、你该选谁**"。篇幅不设限，逐个讲透。
>
> **配套阅读**：路线与选型总纲见 [3.10 主文件](./3.10-computer-browser-use.md)；如何自己从零搭一个见 [3.10 实现深拆](./3.10-implementation-deep-dive.md)。

---

## 怎么看这些产品：四个维度

别被产品名和 benchmark 分数带跑。看任何一个 Computer/Browser Use 产品，抓这四个结构性维度，就能一眼看穿它：

| 维度 | 在问什么 | 取值 |
|---|---|---|
| **技术路线** | 模型"看"什么、"点"什么 | 视觉（截图+坐标）/ 结构（DOM/无障碍树+元素） |
| **产品形态** | 你怎么用它 | 可编程 API/模型 · SDK/框架 · 消费级浏览器 · 通用自主 Agent |
| **差异化机制** | 它独有的关键设计 | 如注入分类器、per-step 安全裁决、out-of-process 隔离、多子 Agent |
| **安全架构** | 被恶意页面攻击时怎么办 | 沙箱/隔离方式、审批机制、权限边界 |

下面按**产品形态**分四类逐个拆。

---

# 一、可编程 API / 模型（开发者搭 Agent 的底座）

这三家是"给开发者调的 computer-use 大模型"，都是**视觉路线**，机制同构（截图→模型出动作→执行→回灌截图），区别在动作集、安全设计、生态与价格。

## 1. Anthropic Computer Use

**一句话**：最通用的"看屏幕、控键鼠"能力，桌面/浏览器通吃，但要你自己搭 loop 和沙箱。

### 是什么
- 一个 API 工具（**beta**，需 beta header，如 `computer-use-2025-11-24` 对应 Claude Sonnet 5 / Opus 4.8 等）。
- Anthropic **只给你模型能力 + 一份 Docker 参考实现**，真正的 agent loop（sampling loop）、虚拟显示器（Xvfb）、动作执行代码要**你自己组装**。这给了最大控制权，也意味着最大工程量。

### 怎么工作（机制）
截图 → 连同任务发给 Claude → Claude 从固定**动作词汇**里选一个返回 → 你的代码在沙箱里执行 → 新截图回灌 → 循环，直到 Claude 不再请求工具（任务完成）或触及 `max_iterations`。动作词汇含 `screenshot / left_click[x,y] / right_click / double_click / mouse_move / left_click_drag / type / key / scroll`，新版还有 **`zoom`**（放大看清小字，因为默认截图分辨率下文件名/标签/行号常糊）。

### 差异化机制
- **内建提示注入分类器**：自动扫描截图，检测到疑似注入指令时，**自动让模型停下、请求用户确认**再继续（可联系官方关闭，但无人值守场景要另想办法）。这是"模型层 + 分类器层"双重防御的范式。
- **支持 ZDR（零数据保留）**：企业合规友好。

### 能力与限制
- 能力：Claude Sonnet 4.6 在 **2026-02 把 OSWorld 推到约 72.5%，逼近人类天花板（~72%）**，是当前最强的通用 computer-use 模型之一。
- 限制：慢（每步 2–5 秒）、贵（每步一次多模态调用）、beta、安全需自建沙箱与最小权限。
- 采用者：Asana、Canva、Cognition、DoorDash、Replit 等。

### 什么时候选它
需要**跨任意桌面应用**（不止浏览器）、要**自己掌控 loop 与环境**做深度定制、且能承担自建沙箱工程量时。要纯网页任务，结构路线（下文 Stagehand）通常更划算。

---

## 2. OpenAI CUA / Operator / ChatGPT agent mode

**一句话**：OpenAI 的 computer-using agent，网页任务尤强，产品形态几经演变，如今是 ChatGPT 里的一个能力。

### 演变史（重要，别被旧名词绕晕）
- **2025-01**：独立产品 **Operator** 上线，背后是 **CUA（Computer-Using Agent）** 模型（GPT-4o 视觉 + 强化学习专训操作 GUI）。
- **2025-08**：Operator **并入 ChatGPT 的 "agent mode"**，独立 Operator 下线；底层模型**升级到 o3 系**。
- 现在：作为 ChatGPT 的 agent 能力存在，也是消费级浏览器 **Atlas**（见第 4 节）的 agent 内核。

### 能力
WebVoyager ~87%（网页导航强项）、WebArena ~58%、OSWorld 早期 ~38.1%（全桌面弱项，说明它偏"网页"而非"全桌面"）。

### 什么时候选它
你已在 OpenAI 生态、要网页导航/操作类任务、且接受"作为 ChatGPT 能力"而非"独立可编程底座"的形态。

## 3. Google Gemini 2.5 Computer Use

**一句话**：Google 把实验产品砍掉、转做"可调 API"的 computer-use 模型，安全设计是它最值得抄的地方。

### 关键转折（行业信号）
Google 于 **2026-05-04 关停实验产品 Project Mariner**，把能力**产品化为可编程 API**——模型 `gemini-2.5-computer-use-preview-10-2025`，经 Gemini API 的 `computer_use` 工具暴露。这被广泛解读为"**行业从实验性独立 Agent 产品，转向 API-first Agent**"的标志。

### 怎么工作（机制）
基于 Gemini 2.5 Pro 的视觉推理，跑在 loop 里：
1. 客户端截图 + 当前 URL，连同**用户指令 + 近期动作历史**发给模型；
2. 模型返回一个 `FunctionCall`（动作）；
3. 客户端在浏览器环境执行，把新截图 + URL 作为 `FunctionResponse` 回传，重启循环，直到任务结束或安全规则叫停。

**支持的动作集**（13 个）：`open_web_browser / search / navigate / click_at / type_text_at / hover_at / scroll_document / scroll_at / key_combination / drag_and_drop / go_back / go_forward / wait_5_seconds`。还**支持并行 function calling**——安全时一轮提议多个原子动作，省往返。

### 差异化机制（最值得学）
**每一步动作都过一个"per-step 安全服务"**，返回 `safety_decision`：
- `regular / allowed`——自动执行；
- `require_confirmation`——**停下，要用户批准**；
- `blocked`——直接拦截。

开发者还能用系统级指令，指定哪些动作必须确认或直接封禁。**这等于把 3.9 的 review/审批做进了模型返回值本身**——审批不再是你在外面补的一层，而是每步动作自带的元数据。任何自建 Agent 都值得学这个设计。

### 价格与落地路径
$1.25 / 百万 input tokens；官方建议**本地 Playwright 环境起步**（最快跑通），生产再评估 Browserbase / Cloud Run。

### 什么时候选它
在 Google 生态、要网页 Agent、且**看重"安全内建到每步"**这套机制时。

---

### 三家可编程方案速对比

| | Anthropic Computer Use | OpenAI CUA / agent mode | Gemini 2.5 Computer Use |
|---|---|---|---|
| 路线 | 视觉（含桌面） | 视觉（网页强） | 视觉（网页为主） |
| 形态 | API 工具，自建 loop/沙箱 | ChatGPT 内能力 | Gemini API `computer_use` 工具 |
| 安全招牌 | 注入分类器 + 自动请求确认 | agent mode 内建安全检查 | **per-step `safety_decision`** |
| 动作特色 | `zoom` 放大看小字 | RL 专训 GUI | 13 动作 + **并行调用** |
| 强项基准 | OSWorld ~72.5%（全桌面强） | WebVoyager ~87%（网页强） | 网页操作，API-first |
| 落地成本 | 高（全自建） | 低（用 ChatGPT） | 中（API + Playwright 起步） |

# 二、消费级 Agentic 浏览器（三家架构对决）

2026 的热点：把浏览器 Agent 做成给普通人用的产品。**这三家能力都不弱，真正拉开差距的是"被恶意网页攻击时的安全架构"**——这也是最值得工程师学习的地方。

## 4. ChatGPT Atlas —— OWL 架构，安全设计的标杆

**一句话**：OpenAI 的 macOS 原生 AI 浏览器（2025-10-21 上线），agent 能力最强，靠 **OWL 架构**把安全做到消费级浏览器里的第一梯队。

### OWL（OpenAI's Web Layer）架构详解
Chromium 当年的革命是"把每个标签页放进独立进程"；**OWL 把这个思路再推一步——把整个 Chromium 浏览器进程移出 Atlas 主应用进程，放进一个隔离的服务层**。

- **异步启动、不阻塞**：Chromium 在后台异步启动，Atlas 不等它，像素几乎即时上屏；Chromium 主线程卡死也不会拖垮 Atlas。
- **Mojo IPC + Swift 绑定**：Atlas 与 Chromium 通过 Chromium 自己的进程间消息系统 **Mojo** 通信，Atlas 客户端用自定义 **Swift 绑定**去操作 Chromium 的 session/profile/webview 等组件。
- **输入事件直达 renderer（关键安全设计）**：**Agent 生成的输入事件直接送到 renderer 进程，不经过特权的 browser 层**——以此保证沙箱隔离，Agent 无法借输入通道去碰特权能力。

### 安全护栏（很全，值得抄清单）
- **禁止 Agent**：在浏览器里执行代码、下载文件、装扩展、访问你电脑上的其他 App 或文件系统；
- **敏感站点暂停**：碰到金融机构等敏感站点会**暂停**，交还控制；
- **会话隔离**：每个 agent 会话**全新开始**，结束时**cookie 与站点数据全部丢弃**；可开多个"登出态"agent 会话，各自独立 tab、互相隔离。

### 已知教训（说明再好的架构也非万能）
2025-10 安全公司 LayerX 披露 **"ChatGPT Tainted Memories"** 漏洞：点一个恶意链接，被污染的页面能把隐藏指令注入 ChatGPT 的**记忆系统**，且**跨会话持续**。这印证了 3.10 主文件与 3.11 的铁律——**提示注入是前沿难题，架构隔离能大幅抬高门槛，但要配合记忆写入的审查、最小权限与人工审批，没有一劳永逸的补丁**。

### 什么时候选它
macOS 用户、要最强 agent 能力、且看重安全架构时（企业评估 AI 浏览器的首选参考）。

---

## 5. Perplexity Comet

**一句话**：免费、全平台的搜索型 agentic 浏览器，胜在易得与覆盖广，安全模型相对更暴露。

- **平台/价格**：Windows / Mac / iOS / Android 全覆盖，浏览器**免费**，agent mode 也免费。
- **能力定位**：把 Perplexity 的搜索型 AI 与完整浏览器能力结合，擅长跨多站点做研究、综合信息。
- **架构/安全**：基于**浏览器扩展**模型——扩展要往页面注入、能触达页面上下文，**攻击面相对 Atlas 的 out-of-process 模型更大**。选它要更谨慎地对待敏感操作。

### 什么时候选它
个人研究、跨端、预算敏感、任务风险较低时。

---

## 6. Claude for Chrome

**一句话**：企业保守向的选择——不逼你换掉 Chrome，把浏览器/OS 控制放进你已有的 Claude 账号。

- **形态**：Chrome **扩展**（2026-04 beta，覆盖付费 Claude 层），后端接 Claude。
- **定位**：面向"想要 agentic 能力、但不想换浏览器/换生态"的（偏企业）用户；把控制权放进每个 Claude Pro 账号。
- **权衡**：扩展形态便利，但安全上不如 Atlas 的进程隔离；企业部署要配合权限与审批策略。

### 什么时候选它
已用 Chrome + Claude、想低摩擦加 agentic 能力、且能自行管控风险的团队。

---

### 三家消费级浏览器横向对比

| | ChatGPT Atlas | Perplexity Comet | Claude for Chrome |
|---|---|---|---|
| 平台 | macOS 原生浏览器 | 全平台（Win/Mac/iOS/Android） | Chrome 扩展 |
| 架构 | **OWL：out-of-process Chromium + Mojo IPC** | 浏览器扩展 | 浏览器扩展 |
| 安全 | **最强**：输入直达 renderer、敏感站点暂停、会话隔离 cookie 丢弃 | 相对更暴露（扩展注入模型） | 中（扩展 + 账号权限管控） |
| 价格 | 随 ChatGPT 订阅 | 免费 | 付费 Claude 层 |
| 定位 | 能力+安全标杆 | 免费全平台研究 | 企业保守低摩擦 |

> **核心洞察**：agentic 浏览器的成败**很大程度取决于安全架构，而不只是能力强弱**。out-of-process 隔离（Atlas）> 扩展注入（Comet/Claude for Chrome）在被攻击时的稳健性。评估任何 AI 浏览器，先问"它被恶意页面劫持时会发生什么"。

# 三、框架 / SDK（结构路线的落地工具）

上面是"模型/产品"，这一类是"帮你把网页 Agent 搭起来的工程工具"，多为**结构路线**（DOM / 无障碍树），可靠、省、快。

## 7. Stagehand（Browserbase）

**一句话**：TypeScript 的 AI 网页 Agent SDK，无障碍树/CDP 路线的代表，网页 Agent 首选之一。

- **机制**：走 **CDP（Chrome DevTools Protocol）层**直接读**无障碍树**、DOM 快照、网络事件，绕开 Playwright"测试优先"的自动等待（对自动化只是拖慢）。
- **三个语义 API**：
  - **`act`**——执行自然语言动作（"点击登录按钮"）；
  - **`extract`**——按 schema 抽取结构化数据（一次拉出商品列表，别一步步点）；
  - **`observe`**——观察页面可操作元素，供规划。
- **为什么强**：无障碍树在"信息完整度"和"信噪比"间取得最佳平衡——比原始 DOM 干净、比截图精确、比二者都省 token，因此可靠性通常比视觉路线高 12–17 个百分点。
- **配套**：与 Browserbase 托管运行时无缝搭配。

## 8. Browser Use

**一句话**：流行的开源浏览器 Agent 编排框架（DOM 路线）。

- 轻量、社区活跃，常作为"上层编排层"，下面接一个浏览器运行时。
- **Manus 的浏览器能力就构建在 browser-use 框架之上**（见第 11 节）。
- 适合想要开源、可自托管、快速起步的项目。

## 9. Playwright / Playwright MCP

**一句话**：浏览器自动化的**事实标准**，MCP 版把它协议化给任意模型用。

- **Playwright**：Claude Code、Cursor、Copilot agent mode 需要浏览器时都用它；**手写 Playwright 脚本在基准任务上约 98% 可靠**（远高于 Agent 自由操作），代价是要写、UI 变了要改。
- **Playwright MCP（微软）**：把浏览器控制**经 MCP 协议**暴露给任意 MCP 客户端，**用无障碍树而非截图**——模型处理结构化文本。与 3.3 的 MCP 生态即插即用。

## 10. Browserbase

**一句话**：托管的浏览器运行时（不是 Agent，是"跑浏览器的云"）。

- 按分钟计费的托管 Chromium，内建**反爬绕过、验证码求解、住宅代理、会话录制**。
- **经验阈值：自托管浏览器超过约 100 浏览器小时/月，运维成本通常就该换托管了。**

# 四、通用自主 Agent 产品

## 11. Manus —— "沙箱 + 多子 Agent + browser-use + 代码解释器"的样板

**一句话**：一个"恰好会写代码"的通用自主 Agent，browser use 是其核心能力之一，它的架构把本章 3.4/3.5/3.7/3.10 缝成了一个产品。

### 架构拆解
- **用 Claude 做主 LLM**；
- **多子 Agent 并行**：一个专职浏览、一个编码、一个综合——**用"专职子 Agent 分担不同职责"来缓解长任务的上下文溢出**（呼应 3.4 上下文工程 / 3.7 多 Agent）；
- **每会话一个隔离的 Linux 沙箱**，内含**基于 browser-use 框架的完整浏览器** + **预装常用库的 Python 解释器**，共约 **29 个工具**（浏览、编码、数据分析、媒体生成）。

### 它最大的工程价值：公开了一套"上下文工程"实战经验
Manus 团队把踩坑总结成了业界最被传阅的一篇《Context Engineering for AI Agents》，六条核心经验（KV-cache、mask 不删 tool、文件系统即记忆、todo.md recitation、保留错误、避免 few-shot rut）——**这些是搭任何长时运行 Agent 的通用金律**，本文不展开，已在 [3.10 实现深拆](./3.10-implementation-deep-dive.md) 第五节逐条讲透。

### 强项
深度竞品调研、多源数据综合、多步 Web 任务、从零产出 pitch deck / 财务模型等"半天工作量"的成品。

### 什么时候选它
任务**横跨浏览器 + 代码 + 写作**、要"端到端产出成品"、且能接受通用自主 Agent 的成本与不确定性时。（若只是纯代码任务，Claude Code / Cursor 更专；纯网页操作，Stagehand 更轻。）

> **旁注**：同类通用自主 Agent 还有 **Devin**（AI 软件工程师，主打可观察全过程 + 可回退 checkpoint 的 coding Agent），机制上与 3.9 的 show-correct-resume 一脉相承。

# 五、总表与选型指引

| 产品 | 类别 | 路线 | 招牌 | 选它的场景 |
|---|---|---|---|---|
| **Anthropic Computer Use** | 可编程模型 | 视觉(含桌面) | 注入分类器 + OSWorld ~72.5% | 跨任意桌面应用、要自控 loop |
| **OpenAI CUA / agent mode** | 可编程/ChatGPT 能力 | 视觉(网页强) | WebVoyager ~87% | OpenAI 生态、网页导航 |
| **Gemini 2.5 Computer Use** | 可编程模型 | 视觉 | **per-step `safety_decision`** | Google 生态、看重安全内建 |
| **ChatGPT Atlas** | 消费级浏览器 | 视觉+结构 | **OWL out-of-process 隔离** | macOS、要最强 agent + 安全 |
| **Perplexity Comet** | 消费级浏览器 | 视觉+结构 | 免费全平台 | 个人研究、跨端、低风险 |
| **Claude for Chrome** | 消费级浏览器 | 结构+视觉 | 不换 Chrome、企业向 | 已用 Chrome+Claude、低摩擦 |
| **Stagehand** | 框架/SDK | 无障碍树/CDP | `act/extract/observe` | 网页 Agent 默认起点 |
| **Browser Use** | 框架/SDK | DOM | 开源编排 | 开源自托管、快速起步 |
| **Playwright (MCP)** | 库/协议 | DOM/无障碍树 | 事实标准、98% 脚本 | 稳定流程固化 / MCP 生态 |
| **Browserbase** | 运行时 | — | 托管 Chromium + 反爬 | >100 浏览器小时/月 |
| **Manus** | 通用自主 Agent | 综合 | 多子 Agent + 沙箱 | 跨浏览+代码+写作产成品 |

### 选型一句话总结
- **要底层能力自己搭** → 三家可编程模型（看生态与安全偏好）；
- **要给普通人用的浏览器** → Atlas（能力+安全）/ Comet（免费全平台）/ Claude for Chrome（不换 Chrome）；
- **要工程化搭网页 Agent** → Stagehand（首选）/ Browser Use（开源）/ Playwright（固化脚本）+ Browserbase（托管运行时）；
