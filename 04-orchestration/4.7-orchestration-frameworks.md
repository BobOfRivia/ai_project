# 4.7 主流框架对比与选型

> **核心问题**：在 2026 的两强格局下（LangGraph / LlamaIndex Workflows），具体项目该选谁？Burr / Haystack Pipelines / Pocket Flow 这些 niche 备选什么时候用？
> **目标**：能从代码型编排框架中为具体项目选一个，理解每个框架的核心抽象与适用场景。
>
> **与 3.8 的关系**：3.8 从"agent 自主循环"视角看 LangGraph，本节从"workflow 编排能力"视角看；两节互相 see-also。

**展开重点**：
- **LangGraph**：图 + 状态 + **内置 checkpointer** + 内置 HITL（`interrupt()`）+ LangChain 生态绑定；强在持久化与状态管理一站式
- **LlamaIndex Workflows**：event-driven `@step` + Context API；持久化 / HITL 自建；强在与 RAG 链路融合
- **Burr**：状态机模型，轻量，适合"有限状态明确"的业务流程；2026 行业声量已下降
- **Haystack Pipelines**：DAG，NLP/RAG 老牌方案
- **Pocket Flow**：极简图，几百行代码风格
- 选型维度：控制力 / 学习曲线 / 持久化是否内置 / HITL 是否内置 / 调试体验 / 生态 / 与 RAG 链路融合度 / 是否 LangChain 系
- 现实组合：很多生产系统 **LlamaIndex 做 RAG + LangGraph 做编排**
