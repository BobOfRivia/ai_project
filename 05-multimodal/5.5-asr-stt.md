# 5.5 语音识别（ASR / STT）

> **核心问题**：2026 STT 选型怎么做？Whisper / Deepgram Nova-3 / NVIDIA Parakeet / ElevenLabs Scribe v2 / AssemblyAI 各占什么位置？流式 vs 批量怎么选？
> **目标**：能为"通话转写 / 会议纪要 / 语音 agent 输入 / 字幕 / 多语言"在准确率、延迟、价格、自托管能力之间选对方案。

**展开重点**：
- **2026 STT 五大主力**：
  - **Deepgram Nova-3 + Flux**（2026-05）：**语音 agent 默认**；强在低延迟、端点检测（end-of-speech）、连续流；Aura-2 TTS 与之配对成全栈
  - **Whisper Large v3 Turbo**（OpenAI 开源）：自托管 / 离线 / 成本敏感场景；生态最成熟
  - **NVIDIA Parakeet TDT 0.6B v3**：边缘 / 设备端 / 快速听写；ASR 新势力
  - **ElevenLabs Scribe v2**：多语言实时表现强
  - **AssemblyAI**：transcript intelligence（说话人分离、摘要、Topic、Sentiment）打包
  - **gpt-realtime-whisper**（OpenAI，2026-05）：超低延迟流式 STT，与 Realtime API 配对
- **选型维度**：
  - **流式 vs 批量**：Agent / 实时对话必须流式；归档转写可批量更便宜
  - **延迟**：voice agent 通常要 < 300ms 端点 + < 200ms 转写
  - **准确率分语种 / 分领域**：医疗、法律、金融术语；带口音；噪音
  - **说话人分离与时间戳**：会议、播客必备
  - **数据合规**：本地 / 私有部署能力（Whisper / NeMo / Parakeet 强）
  - **价格**：批量便宜（Whisper $0.006/分钟级别），实时贵
- **常见组合**：
  - 语音 agent：Deepgram Nova-3 + Flux（流式 + 端点检测）
  - 自托管 / 私有：Whisper Large v3 Turbo 或 NVIDIA NeMo
  - 多语言客服：ElevenLabs Scribe v2 或 Deepgram
  - 归档分析：AssemblyAI（带 Topic / Sentiment / 说话人）
- **边界**：Voice agent 整体架构与 turn 管理放 5.7；本节只覆盖 STT 单组件
